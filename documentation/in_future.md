# A list of suggestions for future features and improvements

Here is a list of some of the ideas we came up with during the project. Most of these improvements and features were considered, but due to time constraints, we did not have time to do them. This list is meant mainly for the possible next teams continuing this project. If this project will not be picked again, this can still be helpful, in case somebody wants to continue this on their own time.

## Feature ideas for the future
- User can change change their password
- User can recover their password from email if they forgot it
- User can recover their password if they forgot it
- User can remove their account
- User can edit the name of their book club
- User can delete their account
- User can see a list of club members if they are the owner or admin
- User can remove club members if they are the owner or admin
- User can suggest books through helmet integration
- Users can leave clubs
- Users can configure their cycles a lot more <br>
  -> meaning they can decide how many books can be suggested, how voting works, will the book that placed second be automatically proposed to the next cycle etc.
- User can make another club member an admin if the user themselves is an admin
## Improvement suggestions

- Start using playwright to test the application
- "More/ Less" button is hidden if the book has no additional information
- Take GDPR into consideration <br>
  -> necessary if e-mail needs to be validated (e.g. password recovery) or other personal information is collected
- API-validation for all paths
- Remove or move the clubs component in the settings page to more sensible location
- Optimize locations for go to login - and go to registration -buttons for all screens
- Lengthen the expration time for authentication token if user visits the site during that time
